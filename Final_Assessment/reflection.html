Reflection

Long ago, Computers and PCs were only devices for people to surf the Internet. Nowadays, people are increasingly using mobile and smartphones of varying screen sizes that run on different OS, like android, windows and ios. However, the appearance of tablets and smartphones has also brought challenges for businesses to develop web apps that are fit for multiple devices.

Responsive web design describes a web design approach that allows websites and pages to render (or display) on all devices and screen sizes by automatically adapting to the screen, whether itâ€™s a desktop, laptop, tablet, or smartphone(FX,2020). In other words, responsive design let us have one site, but with different elements that respond correctly on devices of various sizes. When we see a website that does not have a responsive design and view it on a different screen, for instance, on a desktop computer, the site might show many columns. But when you consider that same website on a smaller screen, it might force you to scroll horizontally, or elements might look distorted. The impact is also complicated because many tablets can be viewed either in portrait orientation or turned sideways for landscape view. Also, on smartphones screens, websites can be even more challenging to see. Large images may break the layout. Sites can be slow to load on smartphones if they are graphics-heavy(RWD, 2020). In conclusion, with responsive design, the website automatically adjusts based on the user's device.

These days, a conversion-focused stunning design of a static website can be developed from scratch using HTML and CSS. To add dynamic features to it, JavaScript will do. Hence, front end technologies such as JavaScript, React JS, Angular JS or Vue JS are commonly used techniques to create responsive websites. As long as you can determine the platform on which the application/website runs, you can use any technology to make it responsive. CSS and HTML are more convenient and more comfortable to create responsive layouts. Whereas JScript is more functional. We can use Html for structure, CSS for styling and media queries, JQuery (or possibly Javascript) for extra class addition and that kind of thing.
